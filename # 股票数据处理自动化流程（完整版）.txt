# 股票数据处理自动化流程（完整版）

## ⚠️ 强制执行 - 每次处理股票必做！

**问题**：总是会忘记 mapping 行业和概念，导致数据不完整。

**解决方案**：自动化脚本 + 强制检查清单 + 邮件通知

---

## 🔄 完整处理流程（9 步）

### 阶段一：数据获取与提取

#### 步骤 1: 获取文章链接
```bash
# 来源：邮件、微信、RSS 等
# 示例：https://mp.weixin.qq.com/s/xxx
```

#### 步骤 2: 保存 Raw Material（强制！）
```bash
cd /home/admin/openclaw/workspace/stock_db

# 使用 scrapling 抓取微信文章
python3 scripts/scrapling_fetch.py \
    --url "https://mp.weixin.qq.com/s/xxx" \
    --output data/raw_articles/文章标题_日期.txt
```

**输出位置**: `data/raw_articles/`

**命名规范**: `文章标题_YYYY-MM-DD.txt`

#### 步骤 3: LLM 提取股票信息
```bash
# 使用提取规则 v12
python3 scripts/extract_rules_v12.py \
    --input data/raw_articles/文章标题_日期.txt \
    --output data/llm_extract_日期.json

# 或使用 Gateway LLM 提取
python3 scripts/llm_extract_gateway.py \
    --input data/raw_articles/文章标题_日期.txt \
    --output data/llm_extract_日期.json
```

**提取内容**:
- ✅ 股票名称、代码
- ✅ accidents（事故/风险）
- ✅ insights（核心观点）
- ✅ core_business（主营业务）
- ✅ supply_chain（产业链）
- ✅ capacity_data（产能数据）
- ✅ valuation（估值信息）

---

### 阶段二：自动 Mapping 与校验

#### 步骤 4: 自动 Mapping（强制！）
```bash
cd /home/admin/openclaw/workspace/stock_db
python3 scripts/auto_stock_mapper.py \
    --input data/llm_extract_日期.json \
    --output data/llm_extract_日期_mapped.json
```

**自动获取**：
- ✅ 股票代码（如无）
- ✅ 同花顺行业（L1/L2/L3）
- ✅ 所属概念
- ✅ 核心业务
- ✅ 合作伙伴

#### 步骤 5: 校验匹配结果并发送邮件（强制！）
```bash
python3 scripts/check_match_and_notify.py \
    --input data/llm_extract_日期_mapped.json \
    --email 18901700722@163.com
```

**检查内容**:
- ✅ 统计 match_type 分布
- ✅ 识别港股（hk_stock）
- ✅ 识别名称错误（none 类型）
- ✅ 识别新股/数据源问题
- ✅ **发送邮件通知**（如有异常）

**匹配率标准**:
| 成功率 | 行动 |
|--------|------|
| ≥ 80% | ✅ 正常，继续流程 |
| 70-80% | ⚠️ 发送邮件提醒 |
| < 70% | ❌ 发送邮件 + 暂停流程 |

**必须邮件通知的情况**:
1. **港股**（hk_stock）- 港股不在 A 股数据库
2. **名称错误**（none）- LLM 提取的名称不准确
3. **新股**（none）- 最新上市股票未收录
4. **数据源问题**（none）- 个别股票数据缺失

## 🚨 常见错误

### 错误 1: 忘记保存 Raw Material
**现象**: 只有提取结果，没有原始文章  
**原因**: 跳过步骤 2  
**解决**: 强制执行 `scrapling_fetch.py`

### 错误 2: 忘记 mapping
**现象**: 股票无行业、无概念、无代码  
**原因**: 跳过步骤 4  
**解决**: 强制执行 `auto_stock_mapper.py`

### 错误 3: 忘记邮件通知
**现象**: 匹配失败未通知用户  
**原因**: 跳过步骤 5  
**解决**: 强制执行 `check_match_and_notify.py`

### 错误 4: 名称错误
**现象**: `match_type: none`  
**原因**: 提取的名称不准确（如"加仓中芯国际"）  
**解决**: 人工核对，修正名称

### 错误 5: 港股未处理
**现象**: A 股数据库无港股信息  
**原因**: 港股不在 `ths_industry_map.json` 中  
**解决**: 手动补充港股行业信息

## ⚠️ 注意事项

### 1. 港股处理
- 港股代码格式：`01585.HK`
- **自动识别并邮件通知**，不纳入处理流程
- 识别规则：
  - 代码包含".HK"
  - 名称含"控股/集团/国际/香港"且无 6 位 A 股代码

### 2. Raw Material 保存
- 位置：`data/raw_articles/`
- 命名：`文章标题_日期.txt`
- 必须保存，便于后续核对和重新提取

### 3. LLM 提取规则
- v12 规则：支持 accidents, insights, core_business, supply_chain, capacity_data, valuation
- Gateway 模型：gateway/qwen3.5-plus
- 输出格式：JSON

### 4. 邮件通知
- 收件人：18901700722@163.com
- 触发条件：
  - 发现港股（hk_stock）
  - 名称匹配失败（none）
  - 成功率 < 80%
- 使用 imap-smtp-email skill 发送

### 5. 数据更新
- 每周更新一次 `stock_auto_map_db.json`
- 来源：`/home/admin/openclaw/workspace/temp/all_stocks_full.json`
- 行业映射：`ths_industry_map.json`
